# سجل التغييرات — Changelog

## 2026-06-08 — تغيير حجم النافذة بحرّية (Free window resizing — Aurora)

أُضيفت بناءً على طلب المستخدم: «اريد امكانية عمل re-size لنافذة التطبيق بشكل حر».

نافذة Aurora بلا إطار (`frameless`)، لذلك لا توفّر حافة نظام للسحب. أضفنا **مقابض سحب
شفّافة** على الحواف الأربع والزوايا الأربع تُغيّر حجم النافذة مباشرةً عبر السحب، مع احترام
الحدّ الأدنى **980×680** وتثبيت الحافة المقابلة عند السحب من اليسار/الأعلى. لا تغيير على
شكل التصميم ولا على محرّك التحميل، ولا تبعيات جديدة.

### الملفات
| الملف | التغيير |
|------|---------|
| `cinemana/gui_web/webview_host.py` | ثابتا `MIN_W/MIN_H` المشتركان، استيراد `FixPoint`، ودالة `JsApi.win_resize(width, height, fix_east, fix_south)` التي تُفوّض إلى `Window.resize(...)`. |
| `cinemana/gui_web/index.html` | ثمانية عناصر `.rz` (حواف + زوايا) قبل إغلاق `#app`. |
| `cinemana/gui_web/app.css` | تموضع المقابض (position:fixed) ومؤشّرات التحجيم + `body.resizing`. |
| `cinemana/gui_web/app.js` | `wireResizeGrips()`: منطق السحب بـ pointer events مع خنق عبر `requestAnimationFrame` ومكالمة واحدة قيد التنفيذ. |

## 2026-06-08 — حفظ الجلسة + أزرار إيقاف/استئناف عامة (Session persistence + global Pause/Resume)

أُضيفت بناءً على طلب المستخدم: «حفظ البيانات المُدخَلة والمسلسل والتقدّم بحيث يمكن الرجوع
والاستكمال حتى بعد إغلاق البرنامج أو إطفاء الكمبيوتر، مع زر للإيقاف المؤقت وزر للاستئناف».

> ملاحظة: هذه التعديلات في واجهة **Tkinter** الحالية (`cinemana/gui.py`)، التي تعمل بـ
> `python app.py --legacy-ui`. واجهة Aurora الجديدة (`cinemana/gui_web`) لم تُبنَ بعد؛ عند
> بنائها يجب ربطها بنفس الدوال أدناه (كلها مستقلة عن الواجهة وقابلة لإعادة الاستخدام).

### ما الجديد للمستخدم
1. **حفظ تلقائي للجلسة:** يُحفظ الرابط، مجلد الحفظ، الجودة، عدد الاتصالات/الأجزاء، الحلقات
   المختارة، وكامل قائمة المسلسل — في `~/.cinemana/last_session.json`.
2. **استعادة عند إعادة الفتح (بدون إنترنت):** عند فتح البرنامج تظهر قائمة المسلسل والحلقات
   مع **حالة كل حلقة ونسبة ما تم تحميله**؛ الحلقات الناقصة تظهر بحالة **«إيقاف مؤقت»** مع
   نسبتها — «كأنك أعطيت البرنامج إيقافاً مؤقتاً». لا يحتاج الأمر إنترنت لعرض المسلسل والتقدّم.
3. **زرّان عامّان** بجانب «ابدأ التحميل»:
   - **«إيقاف مؤقت ⏸»** — يوقف كل التحميلات النشطة دفعةً واحدة (مع تفريغ آمن إلى `.part`).
   - **«استئناف ▶»** — يكمل الموقوفة أثناء العمل؛ وبعد إعادة فتح البرنامج يكمل الجلسة
     المستعادة من حيث توقّف كل ملف (يجلب روابط موقّعة جديدة تلقائياً ويتابع من إزاحة `.part`).
4. قائمة الزر الأيمن لكل حلقة (إيقاف/استئناف/إلغاء) بقيت كما هي.

### الملفات
| الملف | التغيير |
|------|---------|
| `cinemana/session.py` | **جديد.** حفظ/تحميل لقطة الجلسة (`load_session`/`save_session`/`clear_session`) + تحويل الخطة إلى/من dict (`plan_to_dict`/`plan_from_dict`، `episode_to_dict`/`episode_from_dict`). دفاعي تماماً (لا يرمي استثناءات). منفصل عن `settings.json`. |
| `cinemana/service.py` | إضافة `request_pause_all()` و`request_resume_all()` (تكرران على لقطة `list(self.controls)` وتفوّضان إلى دوال الحلقة الواحدة الموجودة؛ آمنة عند الخمول لأن استئناف الحلقة لا يفعل شيئاً حين `_ready is None`). |
| `cinemana/gui.py` | `_save_session()` (يُستدعى عند الجلب/البدء/تغيير الاختيار/الإيقاف/الاستئناف/الإغلاق)، `restore_session()` (يُستدعى في `run()` بعد بناء النافذة)، إعادة هيكلة `_apply_plan(plan, selected_nbs=None, restore_status=False)` لإعادة بناء الشجرة مع الاختيار المحفوظ + طبقة `_restore_statuses()` التي تقرأ المانيفست وتعرض الحالة والنسبة، زرّا «إيقاف مؤقت»/«استئناف» ومعالِجاهما (`_on_pause_all`/`_on_resume_all`) وآلة تفعيل/تعطيل الأزرار عبر دورة الحياة. |
| `cinemana/i18n.py` | مفاتيح جديدة: `btn_pause`, `btn_resume`, `log_pause_all`, `log_resume_all`, `log_session_restored` (عربي/إنجليزي). |
| `tests/test_session.py` | **جديد.** اختبارات: دورة حفظ/تحميل الجلسة (مفقود/تالف → فارغ، الدمج)، تحويل الخطة إلى/من dict (+ الحالات المشوّهة → None)، `request_pause_all`/`request_resume_all` (+ أمان الخمول). |

### نقاط تقنية مهمة
- **التقدّم بالبايت كان يَنجو أصلاً** من إطفاء الجهاز (المانيفست `.cinemana_state.json` الذرّي +
  استئناف Range من حجم `.part`). الناقص الذي أُصلح هو **حفظ البيانات المُدخَلة + قائمة المسلسل +
  الاختيار** ليُستعاد كل شيء بنقرة واحدة بدل إعادة لصق الرابط وإعادة الجلب وإعادة الاختيار.
- **لا تُخزَّن الروابط الموقّعة** في ملف الجلسة (تنتهي صلاحيتها)؛ تُجلَب طازجة لكل حلقة لحظة
  التحميل عبر `ep.nb` → `api.transcoded_files`. لذلك تكفي الخطة المحفوظة لاستكمال التحميل.
- حالة المانيفست `paused` لا تُطبَّع إلى `pending` عند التحميل؛ التشغيل الجديد يضع لكل حلقة
  `EpisodeControl` جديداً غير موقوف فيتابع من إزاحة `.part` تلقائياً.

### الاختبار
- جميع الاختبارات تنجح: **82 اختبار** (٧٢ سابقة + ١٠ جديدة):
  ```
  python -m unittest discover -s tests
  ```
- استيراد كل الوحدات سليم (لا توجد حلقة استيراد) ونصوص الأزرار تظهر بالعربية بشكل صحيح.

### اختبار يدوي شامل
شغّل `python app.py --legacy-ui` → اجلب مسلسلاً → ابدأ → دع بضع حلقات تُحمَّل جزئياً →
أغلق البرنامج → أعد فتحه: ستجد الرابط/المجلد/الجودة/الاختيار مستعادة، والحلقات الناقصة بحالة
«إيقاف مؤقت» مع نسبتها (بدون إنترنت) → اتصل بالإنترنت → اضغط **«استئناف»** فيتابع من نقطة
التوقف بايتاً ببايت. جرّب أيضاً زرّي **«إيقاف مؤقت»/«استئناف»** أثناء التحميل.

## 2026-06-08 — شريط تقدّم مقسّم على طريقة IDM + حتى 32 جزء (Segmented progress bar + up to 32 parts)

**الطلب:** «أريد شريط تقدّم تُوضع فيه الأجزاء، وإمكانية التجزئة إلى 32 جزءًا إن أمكن، تمامًا مثل IDM.»

ميزتان أُضيفتا دون أي تغيير في منطق التحميل: (1) عرض **تقدّم كل جزء** بصريًا كشريط مقسّم إلى
مربّعات تمتلئ مثل IDM، و(2) رفع عدد الأجزاء لكل ملف إلى **32**.

> ملاحظة: هذه التعديلات في واجهة **Tkinter** الحالية (`cinemana/gui.py`). قنوات الإبلاغ في المُحرّك
> (`on_segments`/`on_segment_progress`) مستقلة عن الواجهة، فتصلح لواجهة Aurora الجديدة عند بنائها.

### ما الجديد للمستخدم
1. **شريط مقسّم لكل تنزيل نشِط:** لوحة جديدة «التنزيلات النشطة» تعرض شريطًا لكل حلقة قيد التحميل،
   مقسّمًا إلى مربّعات (مربّع لكل جزء) تمتلئ بالأحمر يسارًا→يمينًا — تمامًا مثل IDM.
2. **حتى 32 جزء/ملف:** صندوق الأجزاء صار يصل إلى 32 (كان 8). تتحقّق 32 جزءًا فعليًا عند
   **التزامن = 1**؛ وعند تزامن أعلى تُخفَّض تلقائيًا (N=2 → 16، N=3 → 10…).

### الملفات
| الملف | التغيير |
|------|---------|
| `cinemana/service.py` | صنف `Events`: ردّا نداء اختياريّان جديدان `on_segments(nb, seg_totals)` (يُطلَق مرّة واحدة لحظة تقرير التقسيم؛ قائمة بعنصر واحد/حجم 0 = مربّع واحد) و`on_segment_progress(nb, k, done_k)`. `_download_episode` يُمرّرهما موصولَين بـ `ep.nb`. رفع `MAX_CONNECTIONS` 16→**32** وحدّ الأجزاء `min(M,8)`→`min(M,32)`؛ **صيغة التقييد بقيت كما هي** (`if N*M>MAX: M=MAX//N`). |
| `cinemana/segmented.py` | مُعاملان جديدان بلا `nb` (`on_segments`/`on_segment_progress`) في **المسارات الثلاثة**: متعدّد الأجزاء (`_run_segments`: إرسال التخطيط + ملء المربّعات المُستأنَفة مسبقًا قبل الخيوط، ثم لكل مربّع داخل `report()` **خارج** `agg_lock`)، المسار المفرد (`_single`: مربّع واحد كسولًا عند أول دفعة)، والحجم المجهول (مربّع واحد بحجم 0). |
| `cinemana/gui.py` | ودجت `SegmentBar(tk.Canvas)` جديدة (ملء أحمر `#c62828` فوق رمادي؛ المسار الساخن `set_segment` يحرّك مستطيل الملء عبر `coords()` فقط → خالٍ من الوميض حتى 32 مربّعًا). لوحة «التنزيلات النشطة» القابلة للتمرير + دوال الحياة `_seg_add_or_replace/_seg_update/_seg_set_speed/_seg_remove/_seg_clear` و`self.seg_bars`؛ نوعا حدث `"segments"`/`"seg_progress"`؛ إزالة الشريط عند done/error/paused وإعادة إنشائه عند الاستئناف؛ تغذية السرعة من `_set_rate`؛ حماية من السباق. صندوق الأجزاء `to=8`→`to=32`. |
| `cinemana/i18n.py` | مفتاح جديد `active_downloads` (عربي/إنجليزي) + ربطه في `_retranslate`. |
| `tests/test_segmented.py` | صنف `TestSegmentCallbacks` (4 اختبارات): إطلاق التخطيط مرّة واحدة وتقدّم كل مربّع حتى الاكتمال، مسار المربّع الواحد، تجاهُل الخادم لترويسة Range، وإطلاق `on_segments` مرّة واحدة عند الاستئناف مع ملء البايتات المُستأنَفة. |

### نقاط تقنية مهمة
- **خيار «شريط لكل تنزيل نشِط»** (لا شريط داخل كل صف): جدول `ttk.Treeview` لا يستطيع استضافة
  Canvas داخل خلية، فجُمعت الأشرطة في لوحة مكدّسة منفصلة.
- **أمان الخيوط:** خيوط الأجزاء تستدعي `on_segment_progress` خارج `agg_lock` (مثل بقية ردود
  النداء)؛ ردّ نداء الواجهة لا يفعل سوى `queue.put`، فلا يُلمَس أي ودجت من خيط غير خيط الواجهة.
- ⚠️ قد تُبطئ شبكة CDN أو تحجب ~32 اتصالًا متزامنًا لملف واحد؛ آلية إعادة المحاولة تعالج
  الانقطاعات المؤقتة، لكن 8–16 جزءًا غالبًا أفضل من حيث السرعة الفعلية.
